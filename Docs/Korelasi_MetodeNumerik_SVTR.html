2.1 Galat — KORELASI PALING KUAT

Kenapa: CER (Character Error Rate) secara harfiah ADALAH galat. Confidence score yang menolak hasil di bawah ambang = penerapan galat untuk keputusan.
Data: prediksi model vs ground truth (teks asli).
RMSE = sqrt( (1/n) · Σ (x_sejati − x_pendekatan)² )
MAE = (1/n) · Σ |x_sejati − x_pendekatan|
Galat relatif = |sejati − pendekatan| / |sejati| × 100%
CER = edit_distance / jumlah_karakter_ground_truth

2.2 Regresi / Curve Fitting — KUAT

Kenapa: (a) Kurva loss bisa di-fit jadi fungsi matematis. (b) Training NN itu sendiri = regresi (minimasi selisih prediksi vs aktual).
Data: titik (epoch, loss) dari log training.
y = a0 + a1·x
a1 = (nΣxy − ΣxΣy) / (nΣx² − (Σx)²)
R² = 1 − (SS_res / SS_tot)

Hasil nyata (v2): regresi polinomial derajat-3 → R² ≈ 0.87.

2.9 Optimasi Numerik — KUAT (paling inti)

Kenapa: Training = optimasi numerik meminimalkan loss. Optimizer proyek = Adam (di config training).
Data: kurva loss menurun 132.98 → 6.63 (reduksi 95%).
Gradient Descent: θ_baru = θ_lama − α · ∇L(θ)
Adam = momentum + adaptive learning rate
(α = learning rate = 0.0005, Cosine schedule)

Masuk akal: Kurva loss menurun = bukti visual gradient descent bekerja.

2.4 Root Finding — SEDANG (didukung silabus)

Kenapa: Silabus: "pencarian akar berkaitan dengan kondisi gradien nol (stationary point) pada optimasi loss." Cari epoch saat turunan loss = 0.
Data: persamaan regresi loss (dari 2.2).
Biseksi: c = (a+b)/2, cek tanda f(a)·f(c)
Newton-Raphson: x_{n+1} = x_n − f(x_n)/f'(x_n)
Secant: x_{n+1} = x_n − f(x_n)·(x_n−x_{n-1})/(f(x_n)−f(x_{n-1}))

Masuk akal: "Decode grafik loss → persamaan → cari akar turunan = epoch konvergen." Newton-Raphson konvergen tercepat, Biseksi paling stabil.

2.7 Sistem Persamaan Linear — SEDANG (otomatis terpakai)

Kenapa: (a) Regresi least-squares diselesaikan via SPL. (b) Tiap layer NN = transformasi linear Wx + b.
Data: matriks normal equation dari regresi.
A·x = b
Eliminasi Gauss → segitiga atas + substitusi mundur
Dekomposisi LU: A = L·U

2.8 Diferensiasi Numerik — SEDANG

Kenapa: (a) Backpropagation = diferensiasi (chain rule). (b) Turunan kurva loss = laju penurunan (kapan training melambat).
Data: kurva loss per epoch.
Beda maju: f'(x) ≈ (f(x+h) − f(x)) / h
Beda tengah: f'(x) ≈ (f(x+h) − f(x−h)) / 2h
Turunan kedua: f''(x) ≈ (f(x+h) − 2f(x) + f(x−h)) / h²

Masuk akal: Turunan loss → 0 = konvergen. Menyambung ke Root Finding.

2.5 Interpolasi — SEDANG (korelasikan ke PREPROCESSING)

Kenapa: Silabus: interpolasi dipakai di image resizing & warping. Proyek BENAR memakainya: cv2.resize dengan INTER_CUBIC/INTER_AREA.
Data: citra sebelum & sesudah resize (ke tinggi 48px).
Linear: f(x) = f(x0) + (x−x0)·(f(x1)−f(x0))/(x1−x0)
Lagrange & Newton (selisih terbagi)

Masuk akal: Resize citra teks = interpolasi piksel. Ini korelasi paling jujur untuk interpolasi (bukan dipaksakan ke kurva loss).

2.3 Integrasi Numerik — LEMAH (untuk perbandingan model)

Kenapa (tipis): "Area under loss curve" = akumulasi loss. Metrik efisiensi: area kecil = konvergen cepat.
Data: kurva loss dua model (v2 vs v3).
Trapesium: ∫ ≈ (h/2)·[f0 + 2f1 + ... + 2f_{n-1} + fn]
Simpson 1/3: ∫ ≈ (h/3)·[f0 + 4f1 + 2f2 + ... + fn]

Masuk akal: Pakai untuk membandingkan efisiensi v2 vs v3.

2.6 ODE — LEMAH / ABSTRAK

Kenapa (abstrak): Silabus: ODE relevan dengan dinamika pembaruan weight. Penurunan loss dimodelkan peluruhan: dL/dt = −k·L.
Data: kurva loss.
Euler: y_{i+1} = y_i + h·f(x_i, y_i)
Heun: rata-rata slope awal & akhir
RK4: kombinasi 4 slope

Masuk akal: Bandingkan solusi numerik (Euler/RK4) vs data aktual loss.

Alur Elegan: 1 Model → 4 Materi Sekaligus

Dari satu kurva loss model v2:

  1. Regresi (2.2) → fit kurva loss jadi persamaan polinomial
  2. SPL (2.7) → persamaan regresi diselesaikan via Eliminasi Gauss
  3. Diferensiasi (2.8) → turunkan persamaan untuk laju penurunan
  4. Root Finding (2.4) → cari akar turunan = epoch konvergen

+ Galat (2.1) ukur kualitas fitting (RMSE/MAE) & Optimasi (2.9) sebagai konteks. → 6 materi dari 1 model.

Kesimpulan

  1. Metode Numerik bukan tempelan — ia fondasi nyata: training = optimasi, loss = galat, preprocessing = interpolasi.
  2. 7 dari 9 materi bisa dikorelasikan kuat/sedang dengan data satu model (v2).
  3. Korelasi jujur butuh: data nyata + rumus silabus + interpretasi masuk akal.
  4. Perbandingan dua model hanya nilai tambah untuk Galat & Integrasi.
